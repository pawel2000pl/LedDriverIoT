Description

RGB (Red, Green, Blue): This color space represents colors as combinations of red, green, and blue light. Each color channel can have values ranging from 0 to 1, where 0 means no intensity and 1 means full intensity. It's commonly used in digital displays and image processing.

HSV (Hue, Saturation, Value): HSV represents colors based on their hue, saturation, and value. Hue refers to the type of color (e.g., red, blue), saturation represents the intensity or purity of the color, and value represents the brightness of the color. It's often used in graphic design and image editing for its intuitive representation of color.

HSL (Hue, Saturation, Lightness): Similar to HSV, HSL also represents colors based on hue, saturation, and lightness. However, in HSL, lightness replaces value, and it represents the perceived brightness of the color. Lightness ranges from 0% (black) to 100% (white), while hue and saturation remain the same as in HSV. HSL is popular for its simplicity and ease of use in web design and CSS styling.

WiFi

Wi-Fi priority list

Description

The WiFi connection feature of your LED RGB controller operates on a prioritized list basis.

1. Adding Networks: Users can add entries to the list containing the name of the network and its password.

2. Connection Priority: When the device searches for available networks, it tries to connect to the first network on the list. If a connection is not established, it moves to the next network on the list, following the priority order.

3. Fallback Option: If none of the networks on the list are found, the device creates its own network (access point) with the specified name and password below the list. However, if the name field is left blank, it won't create an access point and will instead resume network search after a certain period.

4. Repetition of Algorithm: The device repeats this algorithm each time it fails to establish a connection with a network. If it enters access point mode, it does so only after a restart or manual selection of a network from the list (via the "connect" button).

This approach ensures seamless connectivity by attempting to connect to preferred networks first, with a fallback option to create its own network if necessary.

Users can manage network priorities and configurations efficiently through the provided interface, ensuring reliable and user-friendly WiFi connectivity for the LED RGB controller.

Description

The LED RGB controller you are working on incorporates a feature called "filters," allowing you to adjust the input and output curves for optimal performance. This feature is particularly useful due to two main factors:

1. Human Perception of Brightness: Human vision perceives brightness on a logarithmic scale rather than a linear one. As a result, a linear increase in input (like turning a knob) may not translate into a perceived linear increase in brightness. Filters help adjust the input curve to match human perception, ensuring smoother and more intuitive control over brightness levels.

2. LED Characteristics: Different LED diodes can have varying brightness characteristics. Some LEDs may require more power to achieve the same perceived brightness as others. Filters enable you to tailor the output curve to match the characteristics of the specific LEDs you are using, ensuring consistent and accurate brightness control across different diodes.

By combining several predefined functions, these filters allow you to customize the input curve from potentiometers and the output curve to the LED control transistors. This customization helps achieve the desired brightness levels effectively, accounting for both human perception and LED characteristics.
